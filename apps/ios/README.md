# @miraculturee/ios — Capacitor iOS wrapper

Capacitor shell that wraps the `@miraculturee/web` build and ships it as a native iOS app via Codemagic.

## Architecture

```
apps/web          →  Vite + React web app (single source of truth)
apps/ios          →  this package — Capacitor wrapper
  ├── package.json
  ├── capacitor.config.ts
  ├── www/        →  copied from apps/web/dist at build time (gitignored)
  └── ios/        →  native Xcode project (generated by `cap add ios` on a mac)
codemagic.yaml    →  CI workflow at repo root
```

The iOS app is **not a code fork** of the web app — it builds `apps/web` and wraps the output. Any UI change in `apps/web` automatically flows into the next iOS build. Liquid Glass (iOS 26) aesthetics are applied via CSS utilities in `apps/web/src/index.css` under the `.lq-*` prefix, which degrade gracefully on browsers without `backdrop-filter`.

## One-time mac setup

The `ios/` native Xcode project must be generated **on a macOS machine with Xcode installed**. Run once and commit the result:

```bash
cd apps/ios
pnpm install
pnpm run build:web          # builds @miraculturee/web
pnpm run copy:dist          # copies apps/web/dist → www/
pnpm exec cap add ios       # creates ios/ native project
pnpm run cap:sync           # syncs plugins + web assets
git add ios/ package.json capacitor.config.ts
git commit -m "feat(ios): scaffold Capacitor iOS native project"
git push
```

After this one-time step, Codemagic can build `.ipa` files on every push.

## Local iOS development (mac only)

```bash
cd apps/ios
pnpm run build              # build web + copy dist + cap sync
pnpm run cap:open:ios       # opens Xcode
# Then press Run in Xcode, targeting a simulator or device.
```

### Live reload against the Vite dev server

```bash
# Terminal 1 — run the web dev server, bound to your LAN IP
cd apps/web
pnpm run dev:local --host   # vite on 0.0.0.0:5173

# Terminal 2 — build iOS pointing at the dev server
cd apps/ios
CAP_SERVER_URL=http://192.168.1.12:5173 pnpm exec cap sync ios
pnpm run cap:open:ios
```

Save any file in `apps/web/src/` and the iOS simulator reloads instantly.

## Codemagic build

Configured in `codemagic.yaml` at the repo root. The `ios-release` workflow:

1. Installs pnpm deps
2. Builds `@miraculturee/shared` + `@miraculturee/web`
3. Copies `apps/web/dist` → `apps/ios/www`
4. Runs `cap sync ios`
5. Signs with the App Store distribution cert
6. Archives + exports an `.ipa`
7. Uploads to TestFlight

See `codemagic.yaml` for environment variables and code-signing setup.

## Native plugins enabled

| Plugin | Purpose |
|---|---|
| `@capacitor/haptics` | Tactile feedback on raffle entry / purchase |
| `@capacitor/status-bar` | Tune status bar to the noir theme |
| `@capacitor/splash-screen` | Amber glow launch screen |
| `@capacitor/keyboard` | Web-view keyboard resize handling |
| `@capacitor/preferences` | Encrypted key-value store |
| `@capacitor/push-notifications` | APNs for event drops |
| `@capacitor/browser` | In-app Safari for Stripe checkout |
| `@capacitor/device` | Device info for analytics |
| `@capacitor/app` | Deep-link + lifecycle events |

## Bundle ID & App Store metadata

- Bundle ID: `com.miraculturee.app`
- URL scheme: `miraculturee://`
- Deep-link host: `app.miraculturee.com`

Update these in `capacitor.config.ts` if you rebrand.
